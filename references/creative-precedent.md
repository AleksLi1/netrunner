# Creative Precedent — Per-Project Personal Library

## Purpose

This file accumulates **analogies, inversions, and reframings that have actually worked for this user on this project**. Distinct from the shared `analogy-library.md` (which is curated and general); this file is grown by use.

When Netrunner produces a LATERAL response and the user marks an avenue as having unlocked something ("yes, that analogy worked", "great call", explicit acceptance into the roadmap), the avenue's lineage gets appended here.

Over time, this file becomes a personalized "what kind of cross-domain pollination works for this user on this codebase" — paralleling how `CONTEXT.md` tracks closed paths and exhausted clusters but for *open* paths.

## Loading

This file is loaded alongside `lateral-reframings.md` and `analogy-library.md` when LATERAL mode is active. Entries are weighted higher than `analogy-library.md` entries when both apply — the user has already validated these.

## Format

Each entry is appended below as a discrete record:

```markdown
---
### [YYYY-MM-DD] — [short title of the unlocked move]

**Original problem:** [the user's stated framing at the time]

**Lineage:** [ANALOGY: source domain | INVERSION: constraint | NAIVE: question | RECOMBO: A + B + C]

**The reframing:** [the move that unlocked the thinking]

**What it produced:** [the avenue or design that followed, and the outcome if known]

**When to reach for it again:** [trigger conditions for this user's typical work]
```

## Hygiene

- Entries that prove wrong over time (the unlocked move turned out to be a dead end) get a `**Reconsidered:** [date] — [why]` line, but are not deleted. Even failed precedents are signal.
- After 20+ entries, group by lineage type and surface patterns in a `## Patterns observed` section at the top.
- Auto-prune: entries unused for 12+ months and not referenced by any active work get a `**Stale**` flag but are not deleted.
